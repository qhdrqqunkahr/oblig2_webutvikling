Git/Github

For å forstå hva github er må vi først vite hva git er. Git er et open-source version kontroll system som ble laget av Linus Trovalds som også lagde Linux. Hvorfor bruke et versjon kontroll system? En av grunnene for å bruke dette er at når man jobber med et prosjekt så gjør man konstant endringer i koden dette kan bli brukt helt fra alpha fasen og etter den offisielle ferdige utgaven. Dette er da en måte å kunne lagre alle endringene som blir gjort igjennom utviklings prosessen. Med Github kan da flere utviklere jobbe sammen med det samme prosjektet hvor da hver eneste utvikler kan se alle endringer som blir gjort i koden, laste ned koden, endre den og laste opp de nyeste versjonene. I Github lager du noe som vi kaller "Repository" dette er hvor filene i prosjektet blir lagret. En av de fine tingene med Github er at du kan bruke det som en type CV for tidligere arbeid. Jeg syns at github virker som en veldig fin måte å arbeide på både alene og med andre. Det vriker som det kan ta litt tid å bli vandt til å bruke, men etter dette tror jeg det blir en god og enklere måte å arbeide på. Den funskjonaliteten jeg tror jeg liker best med github er grunnlaget for det akkurat det med versjon kontroll.
